# AI教育推進機構 学生部 公式サイト

このファイルは制作の常設ルールである。作業前に必ず読み、ここに反する出力をしない。
詳細仕様（head テンプレ・OGP・JSON-LD・SEO手順・デプロイ）は `SITE_SPEC.md` を参照する。

## このプロジェクトは何か
- AI教育推進機構 学生部の公式サイト。本部(`ai-ready-education.org`)の**下位ブランド**。
- 目的は**活動紹介・参加のきっかけづくり**。第一の読み手は参加に興味のある学生・一般。提携/取材も歓迎だが**前面には押し出さない**。
- 狙うトーンは「**信頼感のあるプロフェッショナル × 明るく前向き**」。**本部ロゴに合わせた白基調 × 濃紺 × ブルー**。写真を活かした落ち着いたレイアウト（カタリバ/コーポレートサイト参考）で、「AIが量産したテンプレ」に見えないこと。文章は **です・ます**中心で平易に。大げさ・気取った・中二病的な言い回し（例「動いている事実」）は使わない。
- **コンテンツの一次資料は「学生部 行動指針」（2026/7/1 作成のPDF）**。スローガン「AIを活かして、自分を育て、価値を生み、社会に還元する」、3つの行動指針（自分の成長に活かす／社会で価値を生み、正当な対価を得る／仲間と社会に還元する）、基本サイクル Learn→Make→Serve→Earn→Share、行動宣言4項。トップ・about の該当箇所はこれに準拠する。

## スタック（厳守）
- **純粋な HTML / CSS / JS のみ。ビルド工程なし。フレームワーク不使用。**
- GitHub Pages で公開。1ファイル1ページ。共通の `css/style.css` と `js/main.js` を全ページで読む。
- 色・余白・角丸はすべて `:root` の CSS変数経由で使う。直値の手打ち禁止。

## 絶対にやらないこと（最重要）
「AIが量産したテンプレ」に見せないための禁止事項。違反は差し戻し対象。
- ❌ グラデーション背景・放射状ブロブ・斜め clip-path。
- ❌ 光るドット・強い影・hover時の大きな拡大。
- ❌ 大文字・超ワイド letter-spacing ラベルの**多用**（セクション見出しの「大きな青い英字」は公認パターン。それ以外に増やさない）。
- ❌ **アクセント色を増やすこと。アクセント（有彩色）はブルー一色のみ（本部ロゴ準拠 `#1f6ab0`）。**
- ❌ 大げさ・気取った見出し（中二病的な言い回し）。連携/提携を前面に押し出す構成。
- ✅ OK（ポップ化）: 角丸は大きめ（カード 12〜16px、ボタン・バッジはピル型 OK）。明るい水色の面・ブルーの数字や差し色。親しみのある です・ます コピー。ホバー時のアクセント線モチーフは維持。

## ページ構成
```
index.html       トップ（ヒーロー=メインメッセージ+写真 / 行動指針3本柱 / 基本サイクル / 行動宣言 / 最新情報 / 3メニュー）
about.html       学生部とは（概要+写真 / 大切にしている考え方 / 基本サイクル / 目指す未来 / CTA）
activities.html  活動内容（活動レポート=note記事ジャンル別 / 主な活動9カード）
members.html     メンバー紹介（運営メンバー / 役割紹介 / 求める人物像 / CTA）
contact.html     お問い合わせ（参加希望 / 活動の質問 / 外部連携・取材 / フォーム）
```
- ナビ項目は全ページで統一: `学生部とは / 活動内容 / メンバー紹介 / お問い合わせ`。
  ページ追加時は全ファイルのナビ・フッターを更新する。
- メインメッセージ（行動指針スローガン）: 「**AIを活かして、自分を育て、価値を生み、社会に還元する。**」
- note: `https://note.com/ai_re_stu`（footer・news・活動レポートで使用）。
- **設計の第一参考は uniways.co.jp**（ユーザー指定）。次のデザイン言語を守る:
  - 背景に**曲線リボンのモチーフ**（薄水色のカーブ + ブルーの帯。SVG data URI で `.hero::before` / `.section--deco::before`）。
  - **有色面（--bg-secondary）の連続フィールドの上に、白い浮遊パネル（`.panel`・`--radius-lg: 24px`）**でコンテンツを載せる。
  - 見出しは **大きな青い英字 + 短い青下線 + 小さな日本語**（ABOUT/ACCESS式。`.section-label`が主役、`.section-title`=日本語は小さく字間広め）。
  - ヘッダーは**白いピル型ナビ**（右寄せ・中に青ピルのCTA）。モバイルは従来のドロワー。
  - プロセス図（基本サイクル）は**白パネル内の薄水色タイル + 青の点線コネクタ**。
  - **フッターは濃紺（--ink）ベタ**。テキストは --on-dark 系トークン。
- お問い合わせのみブルーのボタン扱いにして主CTAを際立たせる。
- 学習リソース（選書・学習導線）は v1 では `activities.html` 内のセクション。
- 連携/提携の話は前面に出さない。トップには置かず、contact に控えめに添える程度にする。
- ヒーローは**白基調 + 写真のスプリット型**（左テキスト×右写真）+ 背景の曲線モチーフ。
- **写真ポリシー**: フリー素材（Pexels等・商用可・クレジット不要）のみ。**人の顔が写る写真は使わない**（手元・後ろ姿・物のみ可）。`images/photos/` にASCII名で保存し、用途に合わせてCDNの `w=/h=/fit=crop` で縮小してから取り込む。

## インタラクションの統一言語
「ラインがホバーで出現」をサイト全体の統一モチーフにする（色はブルー）。
- カード: ホバーで上部ブルー線 + `translateY(-3px)` + 控えめな影。
- 項目: ホバーで左にブルー縦線（`scaleY`で出現、レイアウトずれなし）。

## 文章・コンテンツの規約
- `lang="ja"`、セマンティック HTML、アクセシビリティに配慮。
- 文章は **です・ます**を基本に、親しみやすく平易に。気取った・大げさな言い回しは避ける。
- ただし**話し言葉・くだけすぎた表現は使わない**。落ち着いた丁寧な書き言葉にする。
  - 避ける例: 「こんなことをしています」「こんな人を待っています」「〜してみませんか？」「大丈夫です」「ぜひ」「お気軽にどうぞ」「ワクワク」など。
  - 置き換え例: 「主な活動」「求める人物像」「ご連絡をお待ちしています」「構いません」「歓迎します」など。
- 事実は正確に。数値・日付・期間は**推測で書かず、ユーザーに確認**して確定する。
- 日付は絶対表記（「昨年」ではなく「2025.08」）。News・実績は新しい順。
- 煽り文は控えめに。英語ラベル + 日本語見出しの組み合わせを基調とする。
- 実績は誇張しない。活動は **「実施中／準備中」を明示**する（正直さ＝信頼）。
